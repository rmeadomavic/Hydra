The web server runs on the port configured in `[web] port` (default `8080`). All API responses are JSON unless otherwise noted.

## Authentication

Control endpoints (POST routes for target lock, strike, vehicle commands, pipeline control, and RF hunt) require bearer token authentication. Include your token in the `Authorization` header:

```
Authorization: Bearer <token>
```

Read-only endpoints (`GET` routes for stats, detections, tracks, and the MJPEG stream) do not require authentication.

### Dashboard and stream

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Operator dashboard |
| `GET` | `/stream.mjpeg` | Live MJPEG video stream with bounding boxes, track IDs, and target lock overlays |

### Pipeline stats

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/stats` | Pipeline stats (FPS, inference time, active tracks, total detections, GPS position, detector engine) |
| `GET` | `/api/config` | Current runtime configuration |

### Detection data

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/detections` | Recent detection log entries |
| `GET` | `/api/tracks` | Currently active tracked objects |

### Configuration

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/config/prompts` | Update detection prompts |
| `POST` | `/api/config/threshold` | Update confidence threshold |

**Example: update confidence threshold**

```json
POST /api/config/threshold
Content-Type: application/json

{
  "threshold": 0.6
}
```

### Target control

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/target` | Current target lock state |
| `POST` | `/api/target/lock` | Lock a track for keep-in-frame |
| `POST` | `/api/target/unlock` | Release target lock |
| `POST` | `/api/target/strike` | Send strike command to navigate toward target |
| `POST` | `/api/vehicle/loiter` | Command vehicle to hold position |

**Example: lock a target**

```json
POST /api/target/lock
Content-Type: application/json

{
  "track_id": 5
}
```

**Example: send a strike command**

```json
POST /api/target/strike
Content-Type: application/json

{
  "track_id": 5,
  "confirm": true
}
```

### RF hunt

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/rf/status` | Current RF hunt state and RSSI readings |
| `POST` | `/api/rf/start` | Start an RF hunt with given parameters |
| `POST` | `/api/rf/stop` | Stop the active RF hunt |

**Example: start a WiFi hunt**

```json
POST /api/rf/start
Content-Type: application/json

{
  "mode": "wifi",
  "target_bssid": "AA:BB:CC:DD:EE:FF"
}
```

### Mission review

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/review` | Post-mission review map page |
| `GET` | `/api/review/logs` | List available detection log files |
| `GET` | `/api/review/log/{filename}` | Parse and return detection data from a log file |

### Pipeline control

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/pipeline/stop` | Gracefully stop the pipeline |
| `POST` | `/api/pipeline/pause` | Pause or resume detection |

**Example: pause detection**

```json
POST /api/pipeline/pause
Content-Type: application/json

{
  "paused": true
}
```
