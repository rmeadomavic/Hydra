Hydra can push detection telemetry directly onto your FPV video feed through the flight controller's onboard analog OSD chip (MAX7456 / AT7456E). No extra hardware, no video passthrough, no added latency.

## What it looks like

On your goggles or monitor, you will see a compact status line:

```
T:3 12fps 35ms LK#5TRK
```

That reads as:

- **T:3** — 3 active tracks
- **12fps** — pipeline running at 12 frames per second
- **35ms** — inference time per frame
- **LK#5TRK** — target lock active on track #5, currently tracking

All composited by the FC's OSD chip with sub-millisecond delay.

## Compatible hardware

<Note>
  The OSD overlay requires a flight controller with an onboard OSD chip. This is a hardware requirement — no OSD chip means no overlay.
</Note>

**Works with:**
- Matek H743
- SpeedyBee F405-Wing
- Any FC with an AT7456E or MAX7456 chip

**Will not work with:**
- Pixhawk 6C (no OSD chip) — use the [web dashboard](/features/dashboard) overlay instead

**Running HDZero?** See the [HDZero OSD setup guide](/setup/hdzero-osd) for MSP DisplayPort wiring and parameters.

## Modes

There are two OSD modes depending on how much setup you want to do:

<Tabs>
  <Tab title="Statustext mode">
    The simple option. Detection info appears in the OSD message panel — the same area that shows ArduPilot status messages.

    **Setup:** Just flip the config flag. No changes needed on the flight controller.

    ```ini
    [osd]
    enabled = true
    mode = statustext
    ```

    | Pros | Cons |
    |------|------|
    | Zero FC-side setup | Shares space with other status messages |
    | Works on any OSD-equipped FC | Less control over display formatting |
  </Tab>
  <Tab title="Named value mode">
    The richer option. Uses a Lua script on the flight controller for a dedicated display with stale-link warnings when Hydra stops sending updates.

    **Setup:** Requires copying a Lua script to the FC and setting a few ArduPilot parameters.

    ```ini
    [osd]
    enabled = true
    mode = named_value
    ```

    | Pros | Cons |
    |------|------|
    | Dedicated display area | Requires Lua script on FC |
    | Stale-link warnings | Needs `SCR_ENABLE` and OSD parameters |
    | Richer formatting | Slightly more setup effort |
  </Tab>
</Tabs>

## Lua script setup (named_value mode)

If you choose `named_value` mode, follow these steps to set up the Lua script on your flight controller:

<Steps>
  <Step title="Copy the Lua script">
    Copy `scripts/hydra_osd.lua` from the Hydra repository to the `APM/scripts/` directory on the flight controller's SD card.
  </Step>
  <Step title="Set ArduPilot parameters">
    Configure the following parameters on the flight controller:

    ```
    SCR_ENABLE = 1
    SCR_HEAP_SIZE = 65536
    OSD_TYPE = 1
    OSD1_ENABLE = 1
    ```
  </Step>
  <Step title="Update config.ini">
    Set the OSD mode to `named_value` in your Hydra configuration:

    ```ini
    [osd]
    enabled = true
    mode = named_value
    ```
  </Step>
  <Step title="Reboot the flight controller">
    Power-cycle the FC so the Lua scripting engine starts and loads the OSD script.
  </Step>
</Steps>

## Configuration

The OSD is configured in the `[osd]` section of `config.ini`:

```ini
[osd]
enabled = false
mode = statustext
update_interval = 0.2
```

| Key | Default | Description |
|-----|---------|-------------|
| `enabled` | `false` | Turn FPV OSD overlay on or off (needs MAVLink + FC with OSD chip) |
| `mode` | `statustext` | `statustext` (simple) or `named_value` (needs Lua script on FC) |
| `update_interval` | `0.2` | Seconds between OSD updates — lower is snappier but chattier on the MAVLink bus |
